# Firebase Services

Lokasi: `lib/config/firebase/`

## Struktur File

| File | Fungsi |
|---|---|
| `firebase_service.dart` | Inisialisasi Firebase Core |
| `firebase_options.dart` | Platform-specific config (generated by FlutterFire CLI) |
| `firebase_messaging_service.dart` | FCM: push notification, token management |
| `remote_config_service.dart` | Remote Config: maintenance mode, feature flags |

---

## 1. Firebase Core (`FirebaseService`)

Singleton static class вҖ” panggil sekali di `main.dart`:

```dart
await FirebaseService.init();
```

Otomatis membaca `DefaultFirebaseOptions.currentPlatform` yang dihasilkan oleh FlutterFire CLI.

---

## 2. Firebase Messaging (`FirebaseMessagingService`)

Permanent GetX singleton yang mengelola **FCM (push notification)**.

### Inisialisasi

```dart
final fcm = Get.put(FirebaseMessagingService(), permanent: true);
await fcm.init();
```

### Fitur

- **Permission request** (iOS) вҖ” otomatis di `init()`
- **Token management** вҖ” `fcmToken` observable, auto-refresh
- **APNs token** вҖ” untuk iOS
- **Foreground/Background/Terminated messages**
- **Topic subscribe/unsubscribe**
- **Routing berdasarkan `type`** dari JSON payload

### Cara Pakai

```dart
final fcm = Get.find<FirebaseMessagingService>();

// Baca token
print(fcm.fcmToken.value); // kirim ke backend

// Subscribe ke topic
fcm.subscribeToTopic('promo');
fcm.unsubscribeFromTopic('promo');

// Force refresh token
final newToken = await fcm.refreshToken();
```

### Alur Notifikasi Masuk

```
Push Notification Masuk
    в”Ӯ
    в”ңв”Җ Foreground  вҶ’ _processMessage() вҶ’ ShowNotificationHelper
    в”ңв”Җ Background  вҶ’ firebaseMessagingBackgroundHandler (top-level)
    в””в”Җ Terminated  вҶ’ _checkInitialMessage() вҶ’ _handleNotificationTap()
    
_processMessage():
    1. Parse title, body dari message.notification
    2. Parse JSON dari message.data
    3. Ambil field "type" вҶ’ _parseNotificationType()
    4. Panggil ShowNotificationHelper.showNotification()
```

### Payload JSON yang Diexpect

```json
{
  "notification": {
    "title": "Pesanan Baru",
    "body": "Pesanan #123 telah dibuat"
  },
  "data": {
    "type": "order",
    "order_id": "123",
    "extra_field": "value"
  }
}
```

Type yang didukung: `order`, `alert`, `system`, `chat`, `payment`, `ticket`, `ads`, `marketing`, `general`.

## 3. Remote Config (`RemoteConfigService`)

Permanent GetX singleton untuk membaca **Firebase Remote Config**.

### Inisialisasi

```dart
final rc = Get.put(RemoteConfigService(), permanent: true);
await rc.init();
```

### Default Keys

| Key | Type | Default | Deskripsi |
|---|---|---|---|
| `maintenance_mode` | `bool` | `false` | Apakah app dalam maintenance |
| `maintenance_message` | `String` | `"Aplikasi sedang..."` | Pesan maintenance |

### Cara Pakai

```dart
final rc = Get.find<RemoteConfigService>();

// Cek maintenance mode (reactive)
Obx(() {
  if (rc.maintenanceMode.value) {
    return MaintenancePage(message: rc.maintenanceMessage.value);
  }
  return HomePage();
});

// Custom key
final featureEnabled = rc.getBool('new_feature_enabled');
final apiVersion = rc.getString('api_version');
```

### Real-time Update

Service otomatis listen ke `onConfigUpdated` вҖ” jika ada update di Firebase Console, value akan ter-sync tanpa perlu restart app.

## Setup Awal

1. Jalankan FlutterFire CLI:
   ```bash
   flutterfire configure
   ```
2. Pastikan `google-services.json` (Android) dan `GoogleService-Info.plist` (iOS) sudah ada
3. Set environment variables di `.env` jika diperlukan
